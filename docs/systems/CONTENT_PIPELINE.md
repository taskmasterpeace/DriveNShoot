# DRIVN — Bulk Content Pipeline (how we build a huge world without hand-making everything)

**Status:** DESIGN (research-backed) · **Created:** 2026-07-04 · **Primary stage:** 5
**Source:** research subagent, 2026-07-04. This is the answer to *"we need a way to bulk-create
content easily in the engine."*

> **Key finding:** we already have the seeds of this. `encounter_director.gd` weighted-spawns
> from typed arrays, `data/vehicles/*.tres` is real data-driven authoring, and
> `tools/pixellab_gen.js` is exactly the JSON-batch→asset shape every step below copies. We
> generalize what exists rather than invent.

---

## The 3-Step Pipeline — DO THIS FIRST (in order; each ships independently)

### STEP 1 — Data schema + spreadsheet→`.tres` stamper (the load-bearing wall)
Everything else writes into this. Build it first.
- Write `Resource` scripts per content type: `DataWeapon`, `DataItem`, `DataEnemy`,
  `DataLootTable`, `DataBuilding`, `DataNPC` (mirror the existing `DataVehicle` pattern +
  `.claude/rules/data-files.md`).
- One `@tool extends EditorScript` reads a **CSV/JSON** and emits `.tres` via
  `ResourceSaver.save()` (pattern: Dewlap Games "Generate Godot Resources From CSV"). Author
  content as spreadsheet rows (fits the `xlsx` skill); `.tres` are build artifacts.
- **Immediately refactor `encounter_director.gd`** to spawn from these Resources instead of
  hardcoded arrays/rolls. **Loot tables become data** (`DataLootTable.roll(rng)`), replacing
  `PICKUP_KINDS` + `randi()`.
- **Seeded persistence:** save the `base_seed:int`, NOT generated bytes. Regenerate the world +
  loot placement on load; keep a small delta dict for player changes (opened caches, killed
  uniques). Tiny saves, consistent world. *(Hard requirement — saves already persist state.)*

### STEP 2 — Claude-as-data-generator, wired to Step 1 (the unfair advantage)
- Define a **JSON Schema per type**. Prompt Claude to emit arrays of *data rows* (not prose):
  50 items, 20 enemy variants, NPC barks, `.dialogue` seeds.
- **Validate/normalize/dedupe** in a script (reject bad enums, clamp numbers) → run the Step-1
  stamper. **Commit the JSON to git for human diff-review** — models author, humans approve,
  **the runtime never sees a model**. Natural overnight `/loop` job.

### STEP 3 — Seeded WFC town assembly on a Kenney kit (the "big world" unlock)
- Drop in **`AlexeyBond/godot-constraint-solving`** (Godot 4 Wave Function Collapse, multithreaded,
  supports **scene tiles** = 3D prefabs, not just 2D).
- Import **Kenney Modular Buildings / Building Kit / City Kit** (CC0, low-poly, commercial-OK) as
  the tile set. Author ~15 town tiles once (gate, market, garage, fuel depot, rubble); WFC (on a
  `WorkerThreadPool`, seeded from `base_seed`) assembles **infinite coherent towns** that
  regenerate identically for saves. Add `MultiMesh` greebling (pipes/barrels/scrap) for density.

Then run the local **image→3D** art track (below) as an ongoing background job.

## The technique menu (effort/payoff, all Godot-4.5-friendly)

**Procedural:** seeded chunk streaming (L/H — extend `road_manager.gd`) · WFC town/junkyard
layouts (M/H — `godot-constraint-solving`) · split-grammar/L-system buildings & road networks
(M/M) · `FastNoiseLite` biome/danger fields (L/M — feed `encounter_director`). *Never generate on
the main thread — use `WorkerThreadPool` or spread across frames.*

**Data-driven (backbone):** `.tres` templates (L/H) · CSV/JSON→`.tres` stamp (L/H) · loot tables
as data (L/H) · seed-not-bytes persistence (L/H).

**AI-assisted (our edge, ~$0 local):**
- **LLM data gen (Claude)** (L/H) — schema-validated item/NPC/dialogue/loot rows, git-reviewed.
- **Local image→3D** (M/H) — **TRELLIS.2** (best topology/PBR, fast — vehicles/machinery),
  **Hunyuan3D-2.1** (most consistent, 8K tex — hero assets), **TripoSR** (MIT, sub-second —
  prototype). Run via ComfyUI → **decimate in Blender** → export **glTF/.glb** (Godot imports
  natively) → wrap in the `VehicleBody3D`/prop scene.
- **AI texture/sprite** (L/H) — keep `tools/pixellab_gen.js` for sprites/portraits; local SD for
  one **shared rust/metal trim-sheet** that skins every building + vehicle → instant art cohesion.
- **Discipline:** AI runs at **author-time, never play-time**. Pipeline = `prompt → generate →
  validate/normalize → human diff → commit`. Extend `tools/smoke_test.sh` with a **content-lint**
  pass (every `.tres` referenced, enums valid, no orphans).

**Modular kit-bashing:** Kenney CC0 kits (L/H) · parts-based assembly — vehicles (chassis+turret+
armor) and buildings (foundation+floors+roof) from a parts bank via data (M/H — *recombine 5
chassis × 4 weapons × 3 armor = 60 enemies from ~12 parts*) · `MultiMesh` greeble (L/M).
