# Game Design Documents

This directory contains detailed design documents for CarWorld game systems.

## Document Standard

Every design doc must contain these 8 sections:
1. **Overview** - One-paragraph summary
2. **Player Fantasy** - Target MDA aesthetics
3. **Detailed Rules** - Precise, implementable specifications
4. **Formulas** - Math with variable definitions and examples
5. **Edge Cases** - Explicit handling of unusual situations
6. **Dependencies** - System interactions (bidirectional)
7. **Tuning Knobs** - Exposed values with safe ranges
8. **Acceptance Criteria** - Testable pass/fail criteria

## Creating New Docs

Use `/brainstorm [feature]` to ideate, then create the full GDD here.
Use the `game-designer` agent for structured design consultation.
